Detailed Description

The binary semaphores are a special case of semaphores. They do not have a counter and are used for a single signal event between two tasks or processes.

Post a binary_semaphore.

This function 'posts' a binary semaphore. The state of the binary semaphore is preserved until any other task calls osal_binary_semaphore_wait or osal_binary_semaphore_trywait.

Wait for a binary_semaphore.

This functions waits on a binary semaphore. If the binary semaphore state is not set it waits until someone calls osal_binary_semaphore_post or the specified timeout to occures. If the state was already set it updates the state and returns immediately.

Parameters
[in]semPointer to osal binary_semaphore structure. Content is OS dependent.
[in]toTimeout.
Return values
OKon success.
OSAL_ERR_TIMEOUTif there was no osal_binary_semaphore_post in the specified timeout.

Wait for a binary_semaphore. (blocking)

This function waits on a binary semaphore for some other task to call osal_binary_semaphore_post. If osal_binary_semaphore_post was called before the state of the binary semaphore is updated and it is immediately returned.
